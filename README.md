## 📋 项目概述

MediBot 是一个基于深度学习和知识图谱的智能医疗问答系统，旨在为用户提供准确、及时的医疗咨询服务。该系统通过整合专业医疗知识库，结合先进的自然语言处理技术，为用户提供智能化的健康咨询、疾病初筛和用药指导等服务。

### 🎯 主要功能

- 智能问诊：根据用户描述的症状进行初步诊断
- 用药指导：提供常见药品使用说明和注意事项
- 健康咨询：解答日常保健、疾病预防等相关问题
- 急救指南：提供常见急救知识和处理方法
- 就医建议：根据病情严重程度给出就医建议

## 🔍 项目背景

随着医疗资源分配不均和基层医疗需求增加的矛盾日益突出，智能医疗问答系统能够：

1. 缓解医疗资源紧张状况
2. 提供 24/7 全天候医疗咨询服务
3. 降低基层医疗机构压力
4. 提高医疗信息可及性
5. 助力医疗资源下沉

## 🛠️ 技术架构

### 核心技术栈

- **后端框架**：Python + FastAPI
- **前端框架**：Vue.js + Element UI
- **数据库**：
  - Neo4j（知识图谱存储）
  - MongoDB（对话历史存储）
  - Redis（缓存）
- **AI 模型**：
  - BERT（语义理解）
  - GPT（回答生成）
- **部署**：Docker + Nginx

## 💡 核心功能实现

### 1. 智能问诊流程

1. **症状收集**：通过多轮对话收集用户症状
2. **实体识别**：识别症状、部位等医疗实体
3. **知识图谱查询**：基于症状进行疾病推理
4. **风险评估**：评估病情严重程度
5. **建议生成**：生成诊疗建议和就医指导

### 2. 知识图谱构建

- 数据来源：医学教材、诊疗指南、药品说明书
- 实体类型：疾病、症状、药品、检查项目等
- 关系类型：症状-疾病、药品-适应症、禁忌等
- 定期更新：通过不同渠道获取的医学相关数据更新医学知识库

### 3. 对话管理

- 上下文管理：保持多轮对话连贯性
- 意图识别：准确理解用户咨询意图
- 话题跟踪：实现多话题智能切换
- 异常处理：应对不明确或危险情况


## 🚀 部署方式

支持以下两种部署方式：

### 方式一：Python 环境部署

1. 克隆项目
```bash
git clone https://github.com/username/medibot.git
cd medibot
```

2. 创建并激活虚拟环境（推荐）
```bash
python -m venv venv
source venv/bin/activate  # Linux/Mac
# 或
.\venv\Scripts\activate  # Windows
```

3. 安装依赖
```bash
pip install -r requirements.txt
```

4. 启动服务
```bash
python app.py
```

### 方式二：Docker 部署

1. 克隆项目
```bash
git clone https://github.com/XK-Karl/MediBot.git
cd medibot
```

2. 构建并启动容器
```bash
docker-compose up -d
```

3. 查看服务状态
```bash
docker-compose ps
```
